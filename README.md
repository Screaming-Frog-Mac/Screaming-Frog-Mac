## Running Your First Crawl

### Basic Configuration

<a href="#">Screaming Frog Mac download</a> and setup:

1. Download the installer from screamingfrog.co.uk for Mac.
2. Install and launch the application.
3. Enter the website's root URL (https://example.com) in the URL field.
4. Click Start. The crawler begins following links from the entry URL.

The Crawl Overview panel shows a live count of URLs discovered, URLs crawled, response codes,
and any blocked or excluded URLs as the crawl progresses. For most sites under 1,000 URLs,
the crawl completes in under five minutes.

### Crawl Configuration

Before starting, configure the crawl parameters to match the audit's scope:

**Crawl depth**: Limit the maximum number of link hops from the starting URL. Depth 3–5
covers the entire structure of most sites; unlimited depth crawls the complete site.

**URL exclusions**: Exclude specific URL patterns (pagination, faceted navigation, admin
paths) from crawling through the Exclude tab to focus the crawl on the content that matters.

**Robots.txt respect**: Configure whether to honor or ignore robots.txt directives. Ignoring
robots.txt is appropriate when auditing content that is blocked from search engines but
accessible to users, or when investigating misconfigured robots.txt blocking.

**JavaScript rendering**: Enable JS rendering in Configuration → Spider → Rendering for
JavaScript-heavy sites.

## SEO Audit Workflow

A comprehensive technical SEO audit with <a href="#">Screaming Frog SEO Spider Mac</a> works
through the following investigation areas:

### 1. Indexability Audit

Filter by the following to identify pages that should be indexed but are not:

View Noindex pages under Response Codes → Noindex. Review whether each noindexed page is
intentionally excluded from search or mistakenly noindexed. Export and cross-reference with
Google Search Console data to identify pages that are ranked (and therefore indexed) despite
a noindex directive — an inconsistency that indicates the noindex may have been added after
the page was indexed.

View Blocked by Robots.txt to identify pages blocked by robots.txt. Cross-reference with
canonical tags — pages blocked by robots.txt cannot pass canonical signals to search engines.

### 2. Internal Link Audit

Export Internal → All Links for a complete internal link map. Common issues visible in this data:

**Nofollow internal links**: Internal links carrying nofollow attributes pass no link equity,
meaning internal pages linked only through nofollow links receive no internal link equity
despite having internal links pointing to them.

**Broken internal links**: Every internal link to a 404 or non-200 URL is a wasted link equity
signal and a poor user experience. The Inlinks tab for any broken URL shows where to fix the link.

**Link distribution**: Pages with very few internal inlinks despite being important content may
rank below their potential because they are not well-connected to the internal link graph.

### 3. Duplicate Content Detection

<a href="#">Screaming Frog Mac</a> near-duplicate analysis identifies pages with substantially
similar content that may be competing against each other in search results:

Near-duplicate URLs are grouped in the Duplicate Content report. Each group shows the pages
with similar content, their canonical tags, and their crawl depth. Duplicate content issues
are resolved through canonical tags, 301 redirects, or content differentiation.

### 4. Page Speed Audit

The Response Time column in the crawl data shows the server response time for each URL.
Filtering by high response times identifies pages or sections of the site with slow server
response that affects both user experience and crawl efficiency.
